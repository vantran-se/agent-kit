Markdown

Use markdown blocks for concise prose, bullets, numbered lists, headings, inline formatting, and quotes.

Markdown block

This paragraph shows bold text, italic text, and inline code. Keep paragraphs short so the generated HTML stays scannable.

  • Bullet items are supported
  • Inline links can point to anchors like Charts
  • Use lists when readers need quick scanning

4

Callouts

Callouts support four tones for different kinds of emphasis.

Info

Use info for neutral context, helpful notes, or reader orientation.

Success

Use success for approved decisions, completed milestones, or recommended paths.

Warning

Use warning for constraints, trade-offs, or things a reader should verify before acting.

Danger

Use danger for high-risk issues such as security concerns, data loss, or irreversible actions.

AWS Infrastructure Example

A realistic sample architecture section using the same visual components for cloud infrastructure documentation.

Availability zones
3
Public and private subnets per AZ
Entry points
2
CloudFront and Application Load Balancer
Compute layer
ECS
Fargate services behind ALB

Multi-AZ PostgreSQL plus S3 assets

Request path

1
User

Browser or mobile client sends HTTPS traffic to the public edge.

2
CloudFront + WAF

Static assets are cached, and WAF blocks common abusive traffic before it reaches origin.

3
ALB

Application Load Balancer terminates traffic for dynamic API requests and routes to ECS.

4
ECS service

Fargate tasks run the application inside private subnets with autoscaling.

5
Data stores

RDS stores relational data, ElastiCache handles hot keys, and S3 stores durable objects.

Network

VPC

One VPC with public subnets for ingress and private subnets for compute, cache, and database tiers.

Security

IAM

Least-privilege task roles, security groups by tier, secrets in AWS Secrets Manager, and WAF at the edge.

Observability

CloudWatch

Service logs, ALB metrics, RDS alarms, dashboards, and distributed tracing for request latency.

Layer AWS services Responsibility

Edge

CloudFront, WAF, ACM

TLS, caching, edge protection, and static delivery.

Ingress

ALB, Route 53

DNS routing, health checks, and dynamic request distribution.

Compute

ECS Fargate, ECR

Container deployment, scaling, and image storage.

Data

RDS PostgreSQL, ElastiCache, S3

Relational storage, cache acceleration, and object storage.

Operations

CloudWatch, X-Ray, Secrets Manager

Logs, metrics, traces, alarms, and secret retrieval.
Production guardrail

Keep ECS tasks, RDS, and ElastiCache in private subnets. Only CloudFront and ALB should be internet-facing.

Illustrative monthly cost split
  1. 1

    Provision foundation

    Create VPC, subnets, route tables, security groups, KMS keys, and baseline IAM roles.

  2. 2

    Deploy application

    Push container image to ECR, create ECS task definition, service, target group, and ALB listener rules.

  3. 3

    Attach data services

    Create RDS, ElastiCache, S3 buckets, secrets, and application environment variables.

  4. 4

    Enable operations

    Add CloudWatch dashboards, alarms, log retention, tracing, backup policies, and runbooks.
