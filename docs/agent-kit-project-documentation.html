Quick Start

Install Agent Kit once, then run the setup commands inside each target repository.

  1. 1

    Clone and enter Agent Kit

    Clone this repository and run commands from the repository root.

  2. 2

    Install global assets

    Run python3 scripts/install.py to copy global commands and register MCP permissions.

  3. 3

    Initialize submodules

    Run python3 scripts/install.py --init-submodule or git submodule update --init --recursive to prepare skill sources.

  4. 4

    Check install status

    Run python3 scripts/install.py --check to verify command and MCP setup.

  5. 5

    Bootstrap a target project

    In another project, run /ak:init-project, then /ak:setup-skills and /ak:setup-custom as needed.

bash
git clone <this-repo>
cd agent-kit
python3 scripts/install.py
python3 scripts/install.py --init-submodule
python3 scripts/install.py --check

How the Pieces Fit Together

Agent Kit separates source assets from installed assets so updates remain traceable.

Asset lifecycle

1
Source

Maintain commands in global/, reusable private assets in custom/, and project-maintenance assets in .claude/.

2
Install

scripts/install.py installs global commands and MCP permissions into the user's Claude configuration.

3
Bootstrap

Target projects use /ak:init-project to generate assistant-facing guidance and settings.

4
Extend

Optional skills, commands, and hooks are installed through /ak:setup-skills and /ak:setup-custom.

Layer Path Role

Global source

global/commands/, global/settings.json

Canonical source for globally installed commands and MCP definitions.

Custom reusable assets

custom/commands/, custom/hooks/, custom/skills/

Personal or team assets installed selectively into target projects.

Skill submodules

skills/claudekit-skills/, skills/anthropics-skills/

External skill libraries; do not edit directly.

Repo-local tooling

.claude/commands/, .claude/skills/

Maintenance commands and skills for this repository only.

Visual docs

docs/raw/, docs/

JSON source plus generated HTML documentation.

Tests and scripts

scripts/, tests/

Installer, project initializer, and validation suites.

Commands, MCP Servers, and Hooks

The main operational surface of Agent Kit: slash commands, MCP servers, custom commands, and safety hooks.

Global command Purpose

/ak:init-project

Per-project setup wizard for assistant docs, hooks, MCP permissions, and GitNexus guidance.

/ak:setup-custom

Install selected commands, skills, and hooks from custom/.

/ak:setup-skills

Install selected community and official Anthropic skills from submodules.

/ak:update

Sync Agent Kit MCP permissions into global ~/.claude/settings.json.
MCP server Package Use when

context7

@upstash/context7-mcp@latest

You need up-to-date library documentation or framework examples.

sequential-thinking

@modelcontextprotocol/server-sequential-thinking

A task needs structured reasoning or multi-step planning.

memory

@modelcontextprotocol/server-memory

Persistent knowledge should be stored or retrieved across sessions.

block-dangerous-bash

PreToolUse

Blocks destructive shell patterns such as force push, hard reset, dangerous deletes, database destruction, kill, and unsafe permissions.

check-secrets

PreToolUse

Blocks writes that appear to contain API keys, access tokens, or private keys.

gitnexus-auto-rebuild

PostToolUse

Runs npx gitnexus analyze after writes and edits so the knowledge graph stays fresh.

Maintenance

Use these rules when changing Agent Kit itself.

Source of truth

Edit global/ or custom/, not installed files under ~/.claude/. Do not edit skills/claudekit-skills/ or skills/anthropics-skills/ because they are Git submodules.

Change Edit Then run

Global commands

global/commands/*.md

/ak:sync-docs, then python3 scripts/install.py

MCP servers or permissions

global/settings.json

/ak:sync-docs, then python3 scripts/install.py or /ak:update

Custom commands

custom/commands/*.md

/ak:sync-docs, then /ak:setup-custom in target projects

Custom skills

custom/skills/<name>/SKILL.md

/ak:sync-docs, then /ak:setup-custom in target projects

Custom hooks

custom/hooks/hooks.json

/ak:sync-docs, then /ak:setup-custom in target projects

Reader Checklist

Use this section to orient quickly before making changes or installing assets.

If you are installing Agent Kit

Run python3 scripts/install.py, initialize submodules, and check status before using commands in target projects.

If you are modifying Agent Kit

Update source files, sync docs, regenerate visual documentation when needed, and run python3 tests/run_all.py.

If you are using it in another project

Run /ak:init-project first, then add skills and custom assets only after reviewing what will be installed.

If docs look stale

Trust source files over generated docs, then run /ak:sync-docs and regenerate HTML from docs/raw/.
